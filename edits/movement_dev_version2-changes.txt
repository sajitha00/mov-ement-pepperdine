Integration 1: Eventbrite API
API DOCS
https://www.eventbrite.com/platform/api
ENDPOINT
GET /events/search/
KEY PARAMETERS
location.address = "Malibu, CA"
location.within = "25mi"
sort_by = "date"
expand = "venue,organizer,ticket_availability"

AUTHENTICATION
OAuth 2.0 (Private Token). Dev must create free account at eventbrite.com/platform

FIELDS TO DISPLAY FROM API RESPONSE
	•	name.text — event title
	•	start.local — date and time
	•	venue.name + venue.address — location
	•	description.text — truncate to 120 characters
	•	url — link to ticket page
	•	logo.url — event image

Integration 2: Ticketmaster Discovery API
API DOCS
https://developer.ticketmaster.com/products-and-docs/apis/discovery-api/v2/
ENDPOINT
GET /discovery/v2/events.json
KEY PARAMETERS
city = "Malibu"
stateCode = "CA"
radius = 25, unit = "miles"
sort = "date,asc"

AUTHENTICATION
API Key — free tier at https://developer.ticketmaster.com/

FIELDS TO DISPLAY FROM API RESPONSE
	•	name — event title
	•	dates.start.localDate + localTime
	•	_embedded.venues[0].name + address
	•	_embedded.attractions[0].name — artist or performer name
	•	url — ticket link
	•	images[0].url — event image

Integration 3: Facebook Events (Meta Graph API)
Facebook's public events API is restricted. Start with Option B and upgrade later.

Option A (Easiest)
Use Facebook oEmbed or manually curate Facebook event links for partner merchants and display as cards.
Option B — Recommended Starting Point
Ask each merchant joining The Movement to share their Facebook Event links. Team manually adds these to the /happening/ CMS and displays them as cards with a "Facebook Event" badge.
Option C — Future Upgrade
Use the Meta Graph API with a Page Access Token to pull events from specific Facebook Pages (Pepperdine-area business pages we follow). Requires Facebook App approval. Upgrade to this as the merchant network grows.

Technical Notes for Dev Team
	•	Build a server-side proxy/middleware (Node.js or PHP) to call the APIs. Do NOT expose API keys in frontend JavaScript.
	•	Cache API responses for 30-60 minutes to avoid rate limits. Use Redis or simple file-based cache.
	•	Deduplicate events that appear across multiple sources — match by event name + date + location.
	•	Add "Powered by Eventbrite" and Ticketmaster attribution badges per their API Terms of Service.
	•	Fallback: if APIs return no results, show static message "Check back soon — events coming!" instead of empty state.
	•	Mobile responsiveness is critical — students will primarily view this on phones.
	•	Add "Submit Your Event" button so merchants can submit events for admin review before they go live.
